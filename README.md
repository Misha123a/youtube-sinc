# Sync Music — приложение для совместного прослушивания YouTube Music

Полный MVP: аккаунты, друзья, приглашение в комнату, поиск треков,
синхронное воспроизведение. Друзья могут подключаться из любого города —
через один общий сервер.

## Структура

```
ytmusic-sync/
├── server/                  # Backend + фронтенд (веб-интерфейс)
│   ├── main.py               # FastAPI: все API-эндпоинты + WebSocket
│   ├── database.py           # SQLite: пользователи, заявки в друзья
│   ├── auth.py                # Хэширование паролей, токены сессий
│   ├── ytmusic_search.py     # Поиск треков через ytmusicapi
│   ├── ws_manager.py          # Управление WebSocket-соединениями и комнатами
│   ├── requirements.txt
│   └── static/                # Веб-интерфейс (открывается и в браузере, и в десктоп-клиенте)
│       ├── index.html
│       ├── style.css
│       └── app.js
└── client/                   # Десктоп-обёртка (по желанию — можно обойтись браузером)
    ├── main.py
    └── requirements.txt
```

## Как это работает

1. **Сервер** — единственная точка, которую нужно один раз запустить
   и (для друзей из других городов) сделать доступной из интернета.
   Он хранит пользователей и заявки в друзья в SQLite, и держит
   WebSocket-соединение с каждым залогиненным пользователем.
2. **Комнаты** существуют только в памяти сервера, пока он запущен —
   создал комнату, получил код, пригласил друзей (или просто скинул
   код) — они присоединяются.
3. **Синхронизация** идёт через официальный YouTube IFrame Player —
   сам плеер и аудиопоток остаются на стороне YouTube, сервер только
   пересылает команды play/pause/seek между участниками комнаты.
4. **Десктоп-клиент** — это просто окно со встроенным браузером,
   которое открывает веб-интерфейс сервера. Реально то же самое можно
   открыть в обычном Chrome по адресу сервера — десктоп-обёртка нужна
   только для "ощущения приложения", а не для функциональности.

## Запуск для локальной проверки (перед тем как выкладывать в интернет)

### 1. Сервер

```powershell
cd server
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000
```

Сразу можно открыть `http://localhost:8000` в обычном браузере —
там уже будет рабочий интерфейс: регистрация, друзья, поиск, плеер.
**Рекомендую сначала потестировать именно так, в браузере** — так
гораздо проще отлаживать (F12 → консоль разработчика покажет ошибки),
чем через десктоп-обёртку.

### 2. Десктоп-клиент (по желанию)

```powershell
cd client
pip install -r requirements.txt
python main.py
```

При первом запуске спросит адрес сервера — впиши `http://localhost:8000`
для локального теста.

## Как проверить с другом (тест на двух окнах/вкладках)

1. Открой `http://localhost:8000` в двух вкладках браузера (или запусти
   десктоп-клиент дважды).
2. В одной вкладке зарегистрируйся как `misha`, в другой — как `friend1`.
3. У `misha`: в поле "Добавить" впиши `friend1` → Добавить.
4. У `friend1`: в разделе "Заявки в друзья" нажми "Принять".
5. У `misha`: "Создать комнату" → появится код комнаты и кнопки
   приглашения друзей → нажми на `friend1`.
6. У `friend1` появится всплывающее приглашение → подтвердить.
7. У `misha`: найди трек через поиск, кликни на результат — у обоих
   должен заиграть один и тот же трек синхронно.

## Чтобы друзья из других городов могли подключиться

Локальный `localhost` видно только на твоём компьютере. Чтобы сервер
был доступен из интернета, есть два пути:

**Вариант А — задеплоить сервер в облако** (рекомендуется для
постоянного использования). Подойдут бесплатные/дешёвые варианты:
Railway, Render, Fly.io, обычный VPS. После деплоя у тебя будет
публичный адрес вида `https://твой-проект.up.railway.app` — его и
нужно будет вписывать в десктоп-клиенте вместо `localhost:8000`.

**Вариант Б — проброс порта (port forwarding)** на домашнем роутере —
быстрее для разового теста, но менее стабильно и требует настройки
роутера + белого IP от провайдера.

Я специально не стал сразу деплоить сервер за тебя — скажи, если
хочешь, чтобы я помог настроить деплой на конкретную платформу
(проще всего показать на Railway или Render).

## Известные ограничения текущей версии (MVP)

- Токены сессий хранятся в памяти сервера — при перезапуске сервера
  все разлогинятся и придётся зайти заново.
- Поиск использует неофициальную библиотеку `ytmusicapi` — она
  стабильно работает, но не аффилирована с Google/YouTube официально.
- Нет отображения "кто сейчас онлайн" вне комнаты — только сама
  комната показывает список участников.
- Комнаты не сохраняются после перезапуска сервера (это осознанное
  упрощение — комнаты задуманы как временные "сессии", а не постоянные
  плейлисты).
