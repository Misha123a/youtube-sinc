# Sync Music 2.1

Веб-приложение для синхронного прослушивания музыки с друзьями через YouTube.

## Новое в 2.1

- медиатека авторизованного YouTube-аккаунта;
- личные плейлисты, понравившиеся видео и подписки;
- персональные рекомендации на основе лайков и подписок;
- Google-аватар в профиле Sync Music;
- общая серверная очередь, которая мгновенно синхронизируется у всех участников;
- умная сортировка очереди: чередование участников, защита от дублей и повторов одного исполнителя;
- голосование за будущие треки;
- исправленный градиентный Play/Pause;
- красивый placeholder вместо пустого квадрата обложки;
- полностью удалена hero-card;
- процедурный фон, который оживает во время воспроизведения и останавливается на паузе.

> YouTube IFrame работает в изолированном cross-origin окне, поэтому браузер не разрешает читать реальную амплитуду аудиопотока. Фон реагирует на состояние плеера, текущую позицию трека и уникальный ритмический паттерн композиции.

## Запуск Windows

```powershell
py -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

Или используй существующий `run.bat`.

## Google / YouTube OAuth

1. В Google Cloud включи **YouTube Data API v3**.
2. OAuth Audience установи в **External**. В режиме Testing добавь себя и друзей в Test users.
3. Создай OAuth Client ID типа **Web application**.
4. Добавь origins:
   - `http://localhost:8000`
   - `http://127.0.0.1:8000`
5. Создай `.env` из `.env.example` и вставь Web Client ID.
6. Scopes:
   - `openid`
   - `email`
   - `profile`
   - `https://www.googleapis.com/auth/youtube.readonly`

Google access token хранится только в `sessionStorage` браузера. Публичное имя и URL аватара сохраняются в `app.db`, чтобы профиль выглядел нормально после обновления страницы.

## Ограничения

Официальный YouTube Data API отдаёт плейлисты владельца, лайки и подписки, но не отдаёт точную главную страницу YouTube Music, Supermix и Quick Picks. Поэтому блок «Для тебя» формируется приложением на основе лайков и подписок.

OAuth не передаёт приложению YouTube Premium и не гарантирует отсутствие рекламы. Каждый участник должен быть авторизован в YouTube в своём браузере; окончательное поведение определяет YouTube.
